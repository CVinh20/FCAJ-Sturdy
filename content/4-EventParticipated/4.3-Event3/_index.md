# Summary Report: "DevOps and Cloud Career Orientation"

### Event Objectives

- Provide orientation for students and beginners who are interested in DevOps, Cloud, and modern software engineering careers.
- Explain what technical foundations should be learned first before moving into specialized cloud or DevOps tools.
- Share market trends about hiring demand and salary ranges for software engineering roles in Vietnam.
- Introduce a practical learning roadmap from curiosity to hands-on labs, portfolio building, and community contribution.
- Encourage students to build small real projects, practice consistently, and learn through experimentation.

### Speakers

- **AWS First Cloud Journey speaker team** - Presenters sharing practical guidance about Cloud, DevOps, career development, and the First Cloud Journey learning path.

### Key Highlights

#### What should you learn first?

- The session emphasized that beginners should start with **fundamentals** before jumping directly into advanced tools.
- Important foundations include:
  - **Linux**: Understand command line, file system, permissions, processes, and basic server operations.
  - **Networking basics**: Learn IP, DNS, HTTP/HTTPS, ports, routing, and how services communicate.
  - **Programming languages**: Focus on practical languages such as Python and Golang for automation, backend, and cloud-native development.
  - **Git and CI/CD**: Understand version control, branching, pull requests, automated build, test, and deployment flows.
  - **Containers**: Learn how applications are packaged and run consistently across environments.

#### Understand how applications run

- A DevOps or Cloud learner should know the full lifecycle of an application, not only how to write code.
- Key concepts include building, testing, deploying, reading logs, managing configuration, and using environment variables.
- Understanding runtime behavior helps engineers debug faster and design more reliable systems.
- This mindset connects software development with infrastructure, operations, and monitoring.

#### Build small projects

- The presenter encouraged learners to build small but complete projects instead of only reading theory.
- A useful practice project should include deployment, automation, monitoring, failure testing, and fixing issues.
- The idea is to **deploy a simple app, automate something, monitor it, break it, and fix it**.
- This approach helps learners develop practical confidence and prepares them for real work situations.

#### DevOps career demand and salary trends

- The session showed that hiring demand for roles such as AI/ML engineer, data engineer, cloud engineer, security engineer, and DevOps engineer has increased strongly in recent years.
- DevOps, Cloud, Security, Data, and AI/ML roles are positioned as high-growth directions in the software engineering market.
- A salary table for Vietnam showed that DevOps, Cloud, Security, Data, and AI/ML roles can have strong salary ranges from junior to lead/expert levels.
- The key message was that market demand is growing, but learners need solid fundamentals and real project experience to take advantage of these opportunities.

#### First Cloud Journey roadmap

- The roadmap presented an eight-step learning journey:
  - **Student Curiosity**: Start with curiosity and the willingness to explore.
  - **First Cloud Journey**: Find a suitable learning environment.
  - **Workshop & Community**: Learn from mentors, peers, and community activities.
  - **Hands-on Labs**: Practice through real lab exercises.
  - **School Projects**: Apply knowledge to realistic problems.
  - **Portfolio**: Show capability through completed work.
  - **AWS Partner**: Solve real-world problems with industry context.
  - **Share Back**: Help the next generation by sharing knowledge and experience.

### Key Takeaways

#### Learning Mindset

- Do not skip fundamentals; strong basics make advanced DevOps and Cloud tools easier to understand.
- Learn by building, deploying, observing, breaking, and fixing systems.
- Career growth is a long journey, so consistency matters more than rushing.
- A portfolio of real projects is an important proof of ability.

#### Technical Foundations

- Linux, networking, programming, Git, CI/CD, and containers form the base for DevOps and Cloud work.
- Understanding how applications run in production is just as important as writing code.
- Logs, configuration, environment variables, and deployment behavior are core operational knowledge.
- Hands-on labs help connect theory with real infrastructure practice.

#### Career Development

- DevOps, Cloud, Security, Data, and AI/ML roles have strong growth potential in Vietnam.
- Salary and demand are attractive, but practical skills and problem-solving ability are the real differentiators.
- Community learning can speed up progress because learners receive feedback, examples, and motivation from others.

### Applying to Work

- Review Linux and networking fundamentals before learning more advanced DevOps tools.
- Practice Python or Golang by writing small automation scripts.
- Use Git regularly and understand CI/CD flows through simple projects.
- Containerize a small web application and deploy it to a cloud environment.
- Add monitoring, logs, and environment-based configuration to practice production thinking.
- Build a portfolio page or GitHub repository that documents completed labs and projects.
- Join workshops and community activities to learn from real examples and share progress.
